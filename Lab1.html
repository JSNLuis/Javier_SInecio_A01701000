¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET Se utiliza para obtener información del servidor, no debe hacer ninguna modificación del servidor solo obtiene la información de este.
HEAD Es parecido al método GET, pero sin el cuerpo de respuesta, o sea, se utiliza para comprobar que el archivo existe o se ha modificado.
POST Es utilizado para enviar información al servidor.
PUT El método PUT es usado para solicitar que el servidor almacene el cuerpo de la entidad en una ubicación específica dada por el URL.
PATCH Es utilizado para aplica modificaciones parciales a un recurso
DELETE Borra un recurso en específico.

404 (Not Found), el servidor no puede encontrar el recurso solicitado por el navegador y no es posible determinar si esta ausencia es temporal o permanente. Indican que se ha producido un error cuyo responsable es el navegador.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

En html 5 se separa todo, los elementos y atributos para formularios en HTML5 proveen un mayor grado de marcado semántico que en HTML4 y eliminan gran parte del tedioso trabajo de programar y diseñar que se necesitaba en HTML4. HTML5 define detalladas reglas de parse, incluyendo un control de errores, para que esta sintaxis sea compatible con las implementaciones más populares.
